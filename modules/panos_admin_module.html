paloaltonetworks.panos.panos_admin module – Add or modify PAN-OS user accounts password.

Note

This module is part of the paloaltonetworks.panos collection (version 2.10.2).

To install it, use: ansible-galaxy collection install paloaltonetworks.panos. You need further requirements to be able to use this module, see Requirements for details.

To use it in a playbook, specify: paloaltonetworks.panos.panos_admin.

New in paloaltonetworks.panos 1.0.0

DEPRECATED

Removed in

version 3.0.0

Why

This module is a subset of ERROR while parsing M('panos_administrator'): parameter ‘panos_administrator’ is not a FQCN‘s functionality.

Alternative

Use ERROR while parsing M('panos_administrator'): parameter ‘panos_administrator’ is not a FQCN instead.

Synopsis

  • PanOS module that allows changes to the user account passwords by doing API calls to the Firewall using pan-api as the protocol.

Requirements

The below requirements are needed on the host that executes this module.

  • pan-python can be obtained from PyPI https://pypi.python.org/pypi/pan-python

Parameters

Parameter

Comments

admin_password

string / required

New password for admin_username user

admin_username

string

Username that needs password change.

Default: :ansible-option-default:`"admin"`

api_key

string

API key that can be used instead of username/password credentials.

commit

boolean

Deprecated

Please use ERROR while parsing M('panos_commit_firewall'): parameter ‘panos_commit_firewall’ is not a FQCN, ERROR while parsing M('panos_commit_panorama'): parameter ‘panos_commit_panorama’ is not a FQCN, ERROR while parsing M('panos_commit_push'): parameter ‘panos_commit_push’ is not a FQCN instead.

Commit changes after creating object. If ip_address is a Panorama device, and device_group or template are also set, perform a commit to Panorama and a commit-all to the device group/template.

Choices:

  • :ansible-option-choices-entry:`false`

  • :ansible-option-choices-entry:`true`

ip_address

string / required

IP address (or hostname) of PAN-OS device being configured.

password

string

Password credentials to use for auth unless api_key is set.

port

string

Port used to connect to the PAN-OS device being configured.

Default: :ansible-option-default:`"443"`

role

string

role for admin user

username

string

Username credentials to use for auth unless api_key is set.

Default: :ansible-option-default:`"admin"`

Notes

Note

  • Checkmode is not supported.

Examples

# Set the password of user admin to "badpassword"
  - name: set admin password
    panos_admin:
      ip_address: "192.168.1.1"
      password: "admin"
      admin_username: admin
      admin_password: "badpassword"
