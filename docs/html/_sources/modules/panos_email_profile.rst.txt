.. _panos_email_profile_module:


panos_email_profile -- Manage email server profiles.
====================================================

.. contents::
   :local:
   :depth: 1


Synopsis
--------

Manages email server profiles.




Requirements
------------
The below requirements are needed on the host that executes this module.

- pan-python
- pandevice >= 0.11.1



Parameters
----------

  name (required, type: str)
    Name of the profile.


  config (type: str)
    Custom config log format.


  system (type: str)
    Custom system log format.


  threat (type: str)
    Custom threat log format.


  traffic (type: str)
    Custom traffic log format.


  hip_match (type: str)
    Custom HIP match log format.


  url (type: str)
    PAN-OS 8.0+

    Custom url log format.


  data (type: str)
    PAN-OS 8.0+

    Custom data log format.


  wildfire (type: str)
    PAN-OS 8.0+

    Custom wildfire log format.


  tunnel (type: str)
    PAN-OS 8.0+

    Custom tunnel log format.


  user_id (type: str)
    PAN-OS 8.0+

    Custom user-ID log format.


  gtp (type: str)
    PAN-OS 8.0+

    Custom GTP log format.


  auth (type: str)
    PAN-OS 8.0+

    Custom auth log format.


  sctp (type: str)
    PAN-OS 8.1+

    Custom SCTP log format.


  iptag (type: str)
    PAN-OS 9.0+

    Custom Iptag log format.


  escaped_characters (type: str)
    Characters to be escaped.


  escape_character (type: str)
    Escape character


  provider (type: )
    A dict object containing connection details.


    ip_address (type: str)
      The IP address or hostname of the PAN-OS device being configured.


    username (type: str, default: admin)
      The username to use for authentication.  This is ignored if *api_key* is specified.


    password (type: str)
      The password to use for authentication.  This is ignored if *api_key* is specified.


    api_key (type: str)
      The API key to use instead of generating it using *username* / *password*.


    port (type: int, default: 443)
      The port number to connect to the PAN-OS device on.


    serial_number (type: str)
      The serial number of a firewall to use for targeted commands. If *ip_address* is not a Panorama PAN-OS device, then this param is ignored.



  ip_address (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

  username (type: str, default: admin)

  state (type: str, default: present)

  vsys (type: str, default: shared)
    The vsys this object belongs to.


  device_group (type: str, default: shared)
    (Panorama only) The device group the operation should target.





Notes
-----

.. note::
   - Panorama is supported.
   - Check mode is supported.
   - PAN-OS connectivity should be specified using *provider* or the classic PAN-OS connectivity params (*ip_address*, *username*, *password*, *api_key*, and *port*).  If both are present, then the classic params are ignored.




Examples
--------

.. code-block:: yaml+jinja

    
    # Create a profile
    - name: Create email profile
      panos_email_profile:
        provider: '{{ provider }}'
        name: 'my-profile'
