panos_bgp_policy_filter -- Configures a BGP Policy Import/Export Rule
=====================================================================

.. contents::
   :local:
   :depth: 1


Synopsis
--------

Use BGP to publish and consume routes from disparate networks.




Requirements
------------
The below requirements are needed on the host that executes this module.

- pan-python can be obtained from PyPI https://pypi.python.org/pypi/pan-python
- pandevice can be obtained from PyPI https://pypi.python.org/pypi/pandevice



Parameters
----------

  state (type: str, default: present)
    Add or remove BGP Policy Filter.

    *state=return-object* is deprecated and will be removed in 2.12.


  commit (type: bool)
    Commit configuration if changed.


  filter_type (required, type: str)
    The type of filter.


  policy_name (type: str)
    The name of the policy object.


  policy_type (required, type: str)
    The type of policy object.


  name (required, type: str)
    Name of filter.


  enable (type: bool, default: True)
    Enable filter.


  address_prefix (type: list)
    List of address prefix strings or dicts with "name"/"exact" keys.

    Using the dict form for address prefixes should only be used with *policy_type=aggregate*.


  match_afi (type: str)
    Address Family Identifier.


  match_as_path_regex (type: str)
    AS-path regular expression.


  match_community_regex (type: str)
    Community AS-path regular expression.


  match_extended_community_regex (type: str)
    Extended Community AS-path regular expression.


  match_from_peer (type: list)
    Filter by peer that sent this route.


  match_med (type: int)
    Multi-Exit Discriminator.


  match_nexthop (type: list)
    Next-hop attributes.


  match_route_table (type: str, default: unicast)
    Route table to match rule.


  match_safi (type: str)
    Subsequent Address Family Identifier.


  vr_name (type: str, default: default)
    Name of the virtual router; it must already exist and have BGP configured.

    See :ref:`panos_virtual_router <panos_virtual_router_module>`.


  provider (type: )
    A dict object containing connection details.


    ip_address (type: str)
      The IP address or hostname of the PAN-OS device being configured.


    username (type: str, default: admin)
      The username to use for authentication.  This is ignored if *api_key* is specified.


    password (type: str)
      The password to use for authentication.  This is ignored if *api_key* is specified.


    api_key (type: str)
      The API key to use instead of generating it using *username* / *password*.


    port (type: int, default: 443)
      The port number to connect to the PAN-OS device on.


    serial_number (type: str)
      The serial number of a firewall to use for targeted commands. If *ip_address* is not a Panorama PAN-OS device, then this param is ignored.



  ip_address (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The IP address or hostname of the PAN-OS device being configured.


  username (type: str, default: admin)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The username to use for authentication.  This is ignored if *api_key* is specified.


  password (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The password to use for authentication.  This is ignored if *api_key* is specified.


  api_key (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The API key to use instead of generating it using *username* / *password*.


  port (type: int, default: 443)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The port number to connect to the PAN-OS device on.


  template (type: str)
    (Panorama only) The template this operation should target. Mutually exclusive with *template_stack*.


  template_stack (type: str)
    (Panorama only) The template stack this operation should target. Mutually exclusive with *template*.





Notes
-----

.. note::
   - Checkmode is supported.
   - Panorama is supported.
   - PAN-OS connectivity should be specified using *provider* or the classic PAN-OS connectivity params (*ip_address*, *username*, *password*, *api_key*, and *port*).  If both are present, then the classic params are ignored.
   - If the PAN-OS to be configured is Panorama, either *template* or *template_stack* must be specified.
