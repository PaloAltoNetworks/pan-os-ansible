panos_application_object -- Create application objects on PAN-OS devices.
=========================================================================

.. contents::
   :local:
   :depth: 1


Synopsis
--------

Create application objects on PAN-OS devices.




Requirements
------------
The below requirements are needed on the host that executes this module.

- pan-python can be obtained from PyPI https://pypi.python.org/pypi/pan-python
- pandevice can be obtained from PyPI https://pypi.python.org/pypi/pandevice



Parameters
----------

  name (required, type: str)
    Name of the tag.


  category (type: str)
    Application category


  subcategory (type: str)
    Application subcategory


  technology (type: str)
    Application technology


  risk (type: str)
    Risk (1-5) of the application


  parent_app (type: str)
    Parent Application for which this app falls under


  timeout (type: int)
    Default timeout


  tcp_timeout (type: int)
    TCP timeout


  udp_timeout (type: int)
    UDP timeout


  tcp_half_closed_timeout (type: int)
    TCP half closed timeout


  tcp_time_wait_timeout (type: int)
    TCP wait time timeout


  evasive_behavior (type: bool)
    Application is actively evasive


  consume_big_bandwidth (type: bool)
    Application uses large bandwidth


  used_by_malware (type: bool)
    Application is used by malware


  able_to_transfer_file (type: bool)
    Application can do file transfers


  has_known_vulnerability (type: bool)
    Application has known vulnerabilities


  tunnel_other_application (type: bool)
    Application can tunnel other applications


  tunnel_applications (type: list)
    List of tunneled applications


  prone_to_misuse (type: bool)
    Application is prone to misuse


  pervasive_use (type: bool)
    Application is used pervasively


  file_type_ident (type: bool)
    Scan for files


  virus_ident (type: bool)
    Scan for viruses


  data_ident (type: bool)
    Scan for data types


  description (type: str)
    Description of this object


  tag (type: list)
    Administrative tags


  provider (type: dict)
    A dict object containing connection details.


    ip_address (type: str)
      The IP address or hostname of the PAN-OS device being configured.


    username (type: str, default: admin)
      The username to use for authentication.  This is ignored if *api_key* is specified.


    password (type: str)
      The password to use for authentication.  This is ignored if *api_key* is specified.


    api_key (type: str)
      The API key to use instead of generating it using *username* / *password*.


    port (type: int, default: 443)
      The port number to connect to the PAN-OS device on.


    serial_number (type: str)
      The serial number of a firewall to use for targeted commands. If *ip_address* is not a Panorama PAN-OS device, then this param is ignored.



  ip_address (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The IP address or hostname of the PAN-OS device being configured.


  username (type: str, default: admin)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The username to use for authentication.  This is ignored if *api_key* is specified.


  password (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The password to use for authentication.  This is ignored if *api_key* is specified.


  api_key (type: str)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The API key to use instead of generating it using *username* / *password*.


  port (type: int, default: 443)
    **Deprecated**

    Use *provider* to specify PAN-OS connectivity instead.

    ------------

    The port number to connect to the PAN-OS device on.


  vsys (type: str, default: vsys1)
    The vsys this object belongs to.


  device_group (type: str, default: shared)
    (Panorama only) The device group the operation should target.


  state (type: str, default: present)
    The state.





Notes
-----

.. note::
   - Panorama is supported.
   - Check mode is supported.
   - PAN-OS connectivity should be specified using *provider* or the classic PAN-OS connectivity params (*ip_address*, *username*, *password*, *api_key*, and *port*).  If both are present, then the classic params are ignored.




Examples
--------

.. code-block:: yaml+jinja

    
    - name: Create custom application
      panos_application_object:
        provider: '{{ provider }}'
        name: 'custom-app'
        category: 'business-systems'
        subcategory: 'auth-service'
        technology: 'client-server'
        risk: '1'

    - name: Remove custom application
      panos_application_object:
        provider: '{{ provider }}'
        name: 'custom-app'
        state: 'absent'
