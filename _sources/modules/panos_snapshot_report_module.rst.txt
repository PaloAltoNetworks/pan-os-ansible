.. Description

- A wrapper around the PAN-OS Upgrade Assurance package.
- This is an 'offline' module, meaning it operates only on available facts. It does not need to connect to a device. It's a wrapper around the \ `SnapshotCompare class <https://pan.dev/panos/docs/panos-upgrade-assurance/api/snapshot_compare/#class-snapshotcompare>`__\ .
- The module takes two snapshots made with \ :ref:`paloaltonetworks.panos.panos\_state\_snapshot <ansible_collections.paloaltonetworks.panos.panos_state_snapshot_module>`\  module, compares them and produces a report in a form of a \ :strong:`dict`\ . Keys in this report match the state areas, values contain comparison details.
- You can limit the report to the state area's you're only interested in. You can also adjust the comparison by excluding some or limiting to particular properties.
- Please refer to package's documentation for \ `syntax <https://pan.dev/panos/docs/panos-upgrade-assurance/configuration-details/#readiness-checks>`__\  and \ `configuration dialect <https://pan.dev/panos/docs/panos-upgrade-assurance/dialect/>`__\ .

Notes

.. note::

   - Check mode is not supported.

.. Seealso


.. Examples

Examples
--------

.. code-block:: yaml+jinja

    
    - name: Compare licenses in two snapshots, skip device's serial number to avoid false-positives
      panos_snapshot_report:
        left_snapshot: '{{ snap_1.response }}'
        right_snapshot: '{{ snap_2.response }}'
        reports:
          - license:
              properties:
                - '!serial'
        register: report
    - name: Print the report to stdout
      ansible.builtin.debug:
        var: report.response
